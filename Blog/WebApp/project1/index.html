Im creating a mobile app and website with the objective of allowing users the ability to catalog video games they have played in the past. Furthermore, users will be afforded the opportunity to review, rate, and discuss the games they love.

Today, I wanted to share my thought process in the initial phases of development for a web/mobile application. Let's dive into the features and the tech stack that will power this platform.


Main Features and Functionalities

  • User Profiles: From personalized avatars to game lists, create a profile that truly represents your gaming journey
  • Game Archiving: Keep track of every game you've ever played, and even those you wish to play in the future.
  • Discussion Forums: Engage in lively discussions on game-specific boards or general gaming topics
  • Game Archiving: Share your thoughts on games, complete with spoiler tags and media uploads.

    • For a more extensive look at the basic features, please take a look at my Design Document


High-Level Architecture

Given the project requirements and considering it's an individual project, I will be working with the following stack to balance the ease of use and scalability:


    Web Frontend

  • React React is a popular JavaScript library for building user interfaces. It's component-based, which promotes reusability and modularity. There's a vast community, which means plenty of resources, tutorials, and third-party libraries

    • Mobile

  • React Native With React Native, I can use the same (or very similar) codebase for both iOS and Android apps. Because I'm already using React for the web frontend, much of my knowledge will transfer directly to React Native

    • Backend

  • Node.js with Express.js Node.js allows me to use JavaScript, the same language as the frontend, which can streamline the development process. Express.js is a minimal and flexible Node.js web application framework, making it easier to set up a server

    • Database

  • MongoDB It's a NoSQL database, which means it's flexible and can evolve with the application. With the Mongoose library in Node.js, working with MongoDB becomes even more straightforward. It's also part of the MERN stack (MongoDB, Express.js, React, Node.js), which is a popular combination

    • Hosting/Deployment

  • Heroku Heroku is beginner-friendly and supports Node.js applications out of the box. It also offers a free tier, which is great for someone like me who is new to this aspect of development

    • Version Control

  • Git with GitHub Git is essential for tracking changes, and GitHub provides a platform for hosting and collaborating (should I expand the team in the future)

Final Thoughts

The MERN stack, combined with React Native, offers a unified development experience, especially for solo developers like me. By using JavaScript across the board, I can ensure a streamlined and efficient workflow, making the development process smoother and more enjoyable.
